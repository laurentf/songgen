<task type="checkpoint:human-action">
  <name>Task 1: Build and push Docker image, configure RunPod endpoint</name>
  <what-to-do>
Build the Docker image, push to a container registry, and configure a RunPod serverless endpoint.

**Step 1 — Build the image:**
```bash
cd /path/to/project
docker build -t songgen-worker:v1 .
```
Expected: Build completes. Model download step (~3GB+ for v2-large) takes 5-15 minutes. Final line: "Successfully built [hash]".

If build fails on `git clone https://github.com/tencent-ailab/SongGeneration.git`:
- Network issue: retry or use `--network=host`
- Repo URL changed: check https://github.com/tencent-ailab/SongGeneration for current URL

If build fails on `huggingface-cli download lglg666/SongGeneration-v2-large`:
- Model may require HuggingFace login: add `RUN huggingface-cli login --token $HF_TOKEN` to Dockerfile (with `--build-arg HF_TOKEN=...`)
- Alternatively: download locally first and COPY into image (slower builds but avoids auth)

**Step 2 — Verify checkpoint structure inside image:**
```bash
docker run --rm songgen-worker:v1 python -c "
import os
paths = [
    '/app/songgeneration_v2_large/config.yaml',
    '/app/songgeneration_v2_large/model.pt',
    '/app/ckpt',
    '/app/SongGeneration/tools/gradio/levo_inference.py',
    '/app/SongGeneration/tools/gradio/levo_inference_lowmem.py',
]
missing = [p for p in paths if not os.path.exists(p)]
if missing:
    print('MISSING:', missing)
else:
    print('All paths present — OK')
"
```
If paths are missing, add the corresponding download steps to Dockerfile before the COPY instructions.

**Step 3 — Push to registry:**
```bash
# Docker Hub example:
docker tag songgen-worker:v1 yourusername/songgen-worker:v1
docker push yourusername/songgen-worker:v1

# Or use RunPod's own registry, GitHub Container Registry, etc.
```

**Step 4 — Create RunPod serverless endpoint:**
1. Go to RunPod Console > Serverless > New Endpoint
2. Container Image: `yourusername/songgen-worker:v1`
3. GPU: A5000 or RTX 3090 (24GB VRAM)
4. Container Disk: 50GB minimum (model weights ~22GB + OS + deps)
5. Environment Variables — add these in the "Environment Variables" section:
   - `SUPABASE_URL` = your Supabase project URL
   - `SUPABASE_KEY` = your Supabase service role key
   - `SUPABASE_BUCKET` = `songs`
   - `USE_LOW_MEM` = `false`
6. Advanced > executionTimeout: set to `3600` (1 hour) — prevents job timeout on long batches
7. Click "Deploy"

**Step 5 — Verify Supabase bucket is public:**
Go to Supabase Dashboard > Storage > Buckets > songs > Make Public (if not already done).
This is required for get_public_url() to return accessible URLs.

**Test 2 — Single French track (Success Criterion #1)**

Pass condition: status=success, URL returns HTTP 200, duration > 0.
